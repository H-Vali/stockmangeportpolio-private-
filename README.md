# AssetPilot

공동 운용 자산을 원화 기준으로 관리하는 정적 웹앱입니다.

## 현재 구현 범위

- 통합 대시보드: 전체 평가금액, 투자원금, 평가손익, 총배당, 예수금, 자산배분, 김프 지표, 투자자 비교
- 투자자별 시트: 투자자 선택 칩, 개인 평가금액, 투자원금, 평가손익, 세금, 배당, 예수금, 보유 종목
- 투자자 추가/삭제: 이름 입력으로 추가, 삭제 확인 후 해당 투자자의 입출금/거래 원장도 함께 삭제
- 입출금 입력: 투자자별 입금/출금 원장 관리, 출금은 예수금 한도 초과 방지
- 매수/매도 입력: 종목, 수량, 체결가, 매입 환율, 현재가, 현재 환율 입력
- 물타기 미리보기: 기존 보유 종목 추가 매수 시 새 평단, 평균환율, 예상손익 표시
- 보유 종목 원장 재생: 매수/매도 거래를 재생해 수량, 평단, 평균환율 자동 산출
- 손익 분해: 주가손익과 환차손익을 원화로 분리 표시
- 브라우저 `localStorage` 저장 및 JSON 내보내기
- GitHub Pages public preview 배포 워크플로우

## 핵심 계산식

- 투자원금 = 입금 - 출금
- 예수금 = 입금 - 출금 - 매수금액 + 매도대금
- 평가금액 = 보유종목 평가액 + 예수금
- 평가손익 = 평가금액 - 투자원금
- 주가손익(KRW) = 수량 x (현재가 - 평단가) x 현재환율
- 환차손익(KRW) = 수량 x 평단가 x (현재환율 - 매입평균환율)
- 물타기 후 평균환율 = (기존 외화원가 x 기존환율 + 신규 외화원가 x 신규환율) / (기존 외화원가 + 신규 외화원가)

## 데이터 구조 메모

- `investors`: 투자자 목록
- `cashflows`: 입금/출금 원장
- `trades`: 매수/매도 거래 원장
- `assetCatalog`: 현재가, 현재환율, 자산 분류, 예상 배당 메타데이터
- `ownerId`: 입출금과 거래가 귀속되는 투자자 ID
